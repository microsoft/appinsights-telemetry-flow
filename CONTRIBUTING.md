# Contributing to Application Insights Telemetry Flow

Thank you for your interest in this project!

## How to Contribute

This project accepts **bug reports and feature requests** via
[GitHub Issues](https://github.com/microsoft/appinsights-telemetry-flow/issues).
Pull requests are not accepted at this time.

### Filing Issues

- **Bug reports** — Describe the environment (OS, PowerShell/Bash version,
  Azure hosting type), the command you ran, and what you expected vs. what
  happened. Include the JSON report file if possible (mask your connection
  string first).
- **Feature requests** — Describe the scenario you're trying to solve and
  why the current behavior doesn't meet your needs.
- **Documentation improvements** — If you find unclear or incorrect
  documentation, open an issue describing the problem and which doc page
  is affected.

### Why No Pull Requests?

This is a single-maintainer diagnostic tool. To keep the codebase consistent
and the security posture tight (the scripts handle connection strings and
make HTTP calls to Azure endpoints), all code changes are made by the
maintainer after evaluating issue reports. This may change in the future.
